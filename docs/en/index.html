AAMS / 1.0

Every agent.
One file.

Never start from zero again. AAMS is a declarative standard that gives AI agents a workspace, a memory, and a rulebook — across every tool, every session, every repo.

See it in action

40 seconds. One file. No setup. Your AI agent remembers everything — across sessions, across tools. That's AAMS: the cookbook for your repo.

The problem

You clone a repo. Your agent starts from scratch. No context, no memory, no agreed-upon structure. Every session it reinvents the wheel. Every tool (Copilot, Cursor, Claude, Codex) needs the same explanation again.

AAMS fixes this with a single file.

  • One manifest — .agent.json at repo root. Every agent reads it. Every tool reads it.
  • One workspace — WORKING/ structure with workpapers, whitepapers, diary, memory.
  • One contract — What the agent may do, what it must not, where it starts.

Start in 10 seconds

bash — the only file you need
curl -sO https://raw.githubusercontent.com/DEVmatrose/AAMS/main/.agent.json
  1. 1
    Drop .agent.json into your repo One curl command. Nothing more.
  2. 2
    Tell your agent: start. "Read .agent.json and execute agent_contract.on_first_entry."

What happens next

The agent bootstraps itself:

  • Creates WORKING/ — Workpapers, whitepapers, diary, memory, guidelines, logs
  • Scans the repo — analyzes the codebase, detects architecture and dependencies
  • Writes the first workpaper — documents what it found, what it will do next
  • Indexes into LTM — builds long-term memory that works across sessions

Verifiable. Every step is a markdown file in the repo. git log shows the history. ltm-index.md shows what the agent knew. No more "who changed this and why?" — you grep the workpaper archive.

Next session? The agent reads its memory, creates a new workpaper, and picks up exactly where it left off. No tool lock-in. Switch from Cursor to Claude Code tomorrow — the context stays.

"I don't need this. I manage context myself."

Imagine your grandmother's cookbook — full of brilliant recipes, handwritten notes, and decades of accumulated wisdom. But no table of contents. No cross-references. No way to find what you need without flipping through the entire book.

AAMS is the cookbook with structure. The discipline your grandmother's cookbook was missing. Table of contents, search, version history — for your repo.

At 5 sessions you remember everything. At 50 you re-decide things from two months ago. At 100 your agent hallucinates solutions to problems that were already solved.

You're not adding overhead. You're removing the overhead you didn't know you had.

AAMS is not a framework

AAMS is not a tool. Not a runtime. Not a framework that needs installation. AAMS is a structured context and decision compiler for agents.

Humans define structure, rules, and context. The agent is a deterministic worker — not an actor. Memory is not something the agent "has" — memory is aggregated, traceable history.

The core loop:

Output → Documentation → Decision → Memory → new Context
